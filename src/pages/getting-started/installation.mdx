# Installation Guide

This guide covers installing all DevGuard components:
- **DevGuard Scanner** - CLI tool for security scanning
- **DevGuard CLI** - Management tool for advanced operations
- **DevGuard Platform** - Backend API and web UI (self-hosted)

## DevGuard Scanner

The scanner is the primary tool for running security scans. It's a standalone binary with no dependencies.

### System Requirements

- **OS**: Linux (x86_64, arm64), macOS (Intel, Apple Silicon), Windows (x86_64)
- **Memory**: Minimum 512MB RAM, recommended 2GB+
- **Disk**: 100MB for scanner, additional space for vulnerability databases
- **Network**: Internet access for vulnerability lookups (or offline mode with local DB)

### Installation Methods

<Tabs items={['Go Install', 'Docker', 'Binary Release', 'From Source']}>
  <Tabs.Tab>
    ### Go Install (Recommended)

    Requires Go 1.21 or later.

    ```bash
    # Install latest version
    go install github.com/l3montree-dev/devguard/cmd/devguard-scanner@latest
    
    # Install specific version
    go install github.com/l3montree-dev/devguard/cmd/devguard-scanner@v1.0.0
    
    # Ensure Go bin is in PATH
    export PATH="$PATH:$(go env GOPATH)/bin"
    ```

    **Verify installation:**
    ```bash
    devguard-scanner --version
    ```

    <Callout type="tip">
      **Tip:** Add the PATH export to your `~/.bashrc` or `~/.zshrc` for permanent access
    </Callout>
  </Tabs.Tab>

  <Tabs.Tab>
    ### Docker

    ```bash
    # Pull latest image
    docker pull ghcr.io/l3montree-dev/devguard-scanner:latest
    
    # Run scanner (mount current directory)
    docker run --rm \
      -v $(pwd):/app \
      ghcr.io/l3montree-dev/devguard-scanner:latest \
      sca
    ```

    **Create convenient alias:**
    ```bash
    # Add to ~/.bashrc or ~/.zshrc
    alias devguard-scanner='docker run --rm -v $(pwd):/app ghcr.io/l3montree-dev/devguard-scanner:latest'
    
    # Now use like normal CLI
    devguard-scanner sca
    ```

    **Docker Compose example:**
    ```yaml
    services:
      scanner:
        image: ghcr.io/l3montree-dev/devguard-scanner:latest
        volumes:
          - ./:/app
        working_dir: /app
        command: sca
    ```
  </Tabs.Tab>

  <Tabs.Tab>
    ### Binary Release

    Download pre-built binaries from [GitHub Releases](https://github.com/l3montree-dev/devguard/releases).

    **Linux (x86_64):**
    ```bash
    curl -L https://github.com/l3montree-dev/devguard/releases/latest/download/devguard-scanner_Linux_x86_64.tar.gz | tar xz
    sudo mv devguard-scanner /usr/local/bin/
    ```

    **macOS (Intel):**
    ```bash
    curl -L https://github.com/l3montree-dev/devguard/releases/latest/download/devguard-scanner_Darwin_x86_64.tar.gz | tar xz
    sudo mv devguard-scanner /usr/local/bin/
    ```

    **macOS (Apple Silicon):**
    ```bash
    curl -L https://github.com/l3montree-dev/devguard/releases/latest/download/devguard-scanner_Darwin_arm64.tar.gz | tar xz
    sudo mv devguard-scanner /usr/local/bin/
    ```

    **Windows:**
    ```powershell
    # Download from https://github.com/l3montree-dev/devguard/releases
    # Extract devguard-scanner.exe
    # Move to C:\Program Files\DevGuard\
    # Add to PATH
    ```

    **Verify installation:**
    ```bash
    devguard-scanner --version
    ```
  </Tabs.Tab>

  <Tabs.Tab>
    ### Build from Source

    Requires Go 1.21 or later.

    ```bash
    # Clone repository
    git clone https://github.com/l3montree-dev/devguard
    cd devguard
    
    # Build scanner
    make devguard-scanner
    
    # Install to system
    sudo cp devguard-scanner /usr/local/bin/
    
    # Or add to PATH
    export PATH="$PATH:$(pwd)"
    ```

    **Verify:**
    ```bash
    devguard-scanner --version
    ```

    **Run directly from source (development):**
    ```bash
    go run ./cmd/devguard-scanner/main.go --help
    ```
  </Tabs.Tab>
</Tabs>

## DevGuard Platform (Self-Hosted)

The platform provides centralized vulnerability management, web UI, and API.

### Prerequisites

- Docker & Docker Compose
- PostgreSQL 14+ (or use provided Docker Compose)
- 2GB+ RAM
- 10GB+ disk space

### Quick Start with Docker Compose

<Steps>

### Step 1: Download Configuration

```bash
# Download docker-compose.yaml
curl -O https://raw.githubusercontent.com/l3montree-dev/devguard/main/docker-compose.yaml

# Download environment template
curl -O https://raw.githubusercontent.com/l3montree-dev/devguard/main/.env.example
mv .env.example .env
```

### Step 2: Configure Environment

Edit `.env` file:

```bash
# Database
POSTGRES_PASSWORD=changeme_secure_password

# DevGuard API
DEVGUARD_SECRET_KEY=generate_random_32_chars_here
DEVGUARD_ADMIN_EMAIL=admin@example.com
DEVGUARD_ADMIN_PASSWORD=changeme_admin_password

# Public URL (for production)
DEVGUARD_PUBLIC_URL=https://devguard.example.com

# OAuth (optional)
GITHUB_CLIENT_ID=your_github_oauth_id
GITHUB_CLIENT_SECRET=your_github_oauth_secret
```

<Callout type="warning">
  **Security:** Change all default passwords and generate a secure secret key:
  ```bash
  # Generate secret key
  openssl rand -hex 32
  ```
</Callout>

### Step 3: Start Services

```bash
# Start all services
docker-compose up -d

# Check status
docker-compose ps

# View logs
docker-compose logs -f devguard
```

### Step 4: Access Platform

- **Web UI:** http://localhost:3000
- **API:** http://localhost:8080
- **Health:** http://localhost:8080/health

**Default credentials:**
- Email: From `DEVGUARD_ADMIN_EMAIL`
- Password: From `DEVGUARD_ADMIN_PASSWORD`

<Callout type="info">
  **First login:** You'll be prompted to change your password.
</Callout>

### Step 5: Create Organization & Project

```bash
# Using the API
curl -X POST http://localhost:8080/api/v1/organizations \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "My Organization",
    "slug": "my-org"
  }'

# Create project
curl -X POST http://localhost:8080/api/v1/organizations/my-org/projects \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "My Project",
    "slug": "my-project"
  }'
```

Or use the web UI to create organizations and projects interactively.

</Steps>

### Production Deployment

For production, additional configuration is recommended:

```yaml
# docker-compose.prod.yaml
services:
  devguard:
    image: ghcr.io/l3montree-dev/devguard:latest
    restart: always
    environment:
      - DEVGUARD_PUBLIC_URL=https://devguard.example.com
      - DEVGUARD_SECURE_COOKIES=true
    labels:
      # Traefik labels for HTTPS
      - "traefik.enable=true"
      - "traefik.http.routers.devguard.rule=Host(`devguard.example.com`)"
      - "traefik.http.routers.devguard.tls.certresolver=letsencrypt"
  
  postgres:
    volumes:
      - postgres_data:/var/lib/postgresql/data
    environment:
      - POSTGRES_PASSWORD=${POSTGRES_PASSWORD}
  
  # Add reverse proxy
  traefik:
    image: traefik:v2.10
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock
      - ./traefik.yaml:/traefik.yaml
      - ./acme.json:/acme.json

volumes:
  postgres_data:
```

See [Production Deployment Guide](/how-to/production-deployment) for complete configuration.

## Kubernetes Deployment

For Kubernetes environments, use Helm:

```bash
# Add Helm repository
helm repo add devguard https://charts.devguard.org
helm repo update

# Install with default values
helm install devguard devguard/devguard

# Or with custom values
helm install devguard devguard/devguard -f values.yaml
```

**Example values.yaml:**
```yaml
ingress:
  enabled: true
  hostname: devguard.example.com
  tls: true

postgresql:
  enabled: true
  auth:
    password: changeme

resources:
  limits:
    memory: 2Gi
    cpu: 1000m
  requests:
    memory: 512Mi
    cpu: 250m
```

## Configuration

### Scanner Configuration

Create `~/.devguard/config.yaml`:

```yaml
# API connection
api:
  url: https://api.devguard.org
  token: ${DEVGUARD_TOKEN}

# Default settings
scanner:
  upload: true
  fail-on:
    critical: true
    high: false
  
# Exclusions
exclude:
  paths:
    - node_modules
    - vendor
    - .git
```

### Environment Variables

```bash
# Scanner
export DEVGUARD_TOKEN="your_api_token"
export DEVGUARD_API_URL="https://api.devguard.org"
export DEVGUARD_ASSET_NAME="my-project"

# CLI
export DEVGUARD_ORG="my-org"
export DEVGUARD_PROJECT="my-project"
```

## Troubleshooting

### Installation Issues

**Problem: `go install` command not found**
```bash
# Install Go first
# Visit https://go.dev/doc/install

# Or on macOS
brew install go

# On Linux (example for Ubuntu)
sudo apt install golang-go
```

**Problem: Binary not in PATH after go install**
```bash
# Add Go bin to PATH
export PATH="$PATH:$(go env GOPATH)/bin"

# Make permanent by adding to ~/.bashrc or ~/.zshrc
echo 'export PATH="$PATH:$(go env GOPATH)/bin"' >> ~/.bashrc
```

**Problem: Docker permission denied**
```bash
# Add user to docker group (Linux)
sudo usermod -aG docker $USER
newgrp docker
```

**Problem: Platform won't start**
```bash
# Check logs
docker-compose logs devguard

# Common issues:
# 1. Database not ready - wait 30 seconds and retry
# 2. Port conflicts - change ports in docker-compose.yaml
# 3. Missing environment variables - check .env file
```
